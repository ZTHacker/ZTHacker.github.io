搭建个人博客网站Linux、Github、Hexo

搭建环境：Centos7、github、hexo、Git

1、了解Hexo

Hexo是高效的静态站点生成框架，它基于Node.js。 通过 Hexo 你可以轻松地使用 Markdown 编写文章，除了 Markdown 本身的语法之外，还可以使用 Hexo 提供的标签插件来快速的插入特定形式的内容，而且相对于其他框架，Hexo在速度上也有很大优势。


2、搭建Node.js环境


我们了解到Hexo基于Node.js的，那么我们搭建博客网站首先需要安装Node.js环境。 Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行环境，可以在非浏览器环境下，解释运行 JS 代码。


3、先下载git和npm


Git是目前世界上最流行的分布式版本控制系统，是的，没有之一。使用Git可以帮助我们把本地的网页和文章等内容提交到Gihub上，实现同步。


npm是nodejs的包管理器,所以安装npm的时候会下载nodejs.


yum install git npm

4、下载hexo-cli


npm install hexo-cli -g

-g=-global这里不加-g会安装到当前目录的node_modules文件夹下


5、下载hexo初始文件


相当是一个命令行应用,这里是网站的文件夹,dirname是文件夹名自己换一个有意义的名字


hexo init [dirname]

下面来测试本地博客站点，在本地博客根目录下使用控制台命令：


hexo g  //g是generetor的缩写，生成博客
hexo s  //s是server的缩写，启动服务

此时打开浏览器，输入 http://localhost:4000/，我们将会看到Hexo自带默认主题显示的博客。


6、开启GitHub Pages服务


搭建我们的个人博客需要一个唯一的域名，当然我们可以申请购买一个域名来使用，但是在不是太必要的情况下，我们也可以通过Github Pages获得一个免费使用的域名，这需要我们在Github上新建一个仓库，如下：



这个过程和建立普通的仓库没什么区别，关键在于新仓库的名字，一定要是UserName+“github.io”的形式。这也是之前强调的要起一个好的用户名的原因。这样之后我们最后的博客网站的链接就会是：https://UserName.github.io的形式。
注意：固定新仓库的名字格式并非必须，只是这样操作生成的博客域名比较短小简洁，另起他名生成博客域名会很冗长。


点击Create Repository之后，随后选择Setting进入设置，找到Github Pages如下：



这里我们需要点击Choose a theme任意选择一个选择主题，然后界面会跳转到仓库，会得到如上图的一个域名，这样就可以用这个域名访问自己的博客了。


7、将博客部署到github


在本地执行如下命令，第一个填写自己的github账户名称，第二个填写自己注册github时的邮


git config –global user.name “user.name”
git config –global user.email “user.email”

生成 ssh 密匙

ssh-keygen -t rsa -C user.email

生成的密钥默认在 ~/.ssh/id_rsa.pub


cat ~/.ssh/id_rsa.pub 注意是id_rsa.pub 这个是公钥,id_rsa是私钥
复制pub文件内的所有字符然后到https://github.com/settings/keys添加公钥



8、同步Github,允许公共访问


在本地我们已经搭建了博客，但是还只能自己本地访问。若要别人也能看到，那就需要我们将其同步部署到GitHub上了。还记得我们之前准备的Github仓库吗，这里就要用到了。
首先找到我们的博客仓库，并拷贝仓库地址：

最后执行控制台命令：


npm install hexo-deployer-git —save //安装部署插件
hexo d //部署到github

现在，我们再次访问链接：https://zthacker.github.io，就会发现这里的界面和本地的一样了。如此一来我们搭建的个人博客网站就基本完成了。


9、发布博客


激动人心的时候到了，终于可以发布自己的第一篇博客了。来尝试一下以下的步骤：
在本地博客文件夹根目录输入：



hexo new “我の个人第一篇博文”
hexo g   //生成网页
hexo d  //部署到远端(github)

现在打开我们的博客网站：https://zthacker.github.io



这就是我们发布博客的方法啦，当然这里是为了演示，真正做的时候我们要找到_posts里的博客原文，先编辑内容，然后再部署到github上。


10、更换主题


为了让它看起来更美观一些，我们可以为其更换主题


可以去https://hexo.io/themes/下载我们需要的主题



这些主题都是在github上可以用git下载的，所以我们在本地切换到themes目录下



然后用git clone 加上主题下载地址，这样就能下载到themes下 了，然后在改_config.yml文件，搜索theme字段，并将其值修改为刚才下的主题名字。然后


在控制台下输入如下命令：



hexo clean  //清理缓存
hexo g    //重新生成博客代码
hexo d   //部署到本地




这样我们主题就生效了。


说明一下：在Hexo中有两份主要的配置文件，其名称都是 _config.yml。 其中，一份位于站点根目录下，主要包含 Hexo本身的配置；另一份位于主题目录下，这份配置由主题作者提供，主要用于配置主题相关的选项。所以主题上传成功后可以修改一下站点配置那就需要更改这俩配置文件。写博客时需要用markdown语法，这需要我学习使用markdown了。
